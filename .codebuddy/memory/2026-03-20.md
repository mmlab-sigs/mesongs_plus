## RAHT 分块策略分析（已完成，结论：无效）

### 实验方法
- 脚本: `experiments/analyze_raht_block_stats.py`
- 直接加载 ckpt → RAHT 变换 → 对比均匀/层级/层级+空间三种分块的 AC 系数统计

### 关键结论
- **RAHT 各层 AC 系数统计特性几乎没有差异**：所有层 std 在 1~2 范围内，没有"高层方差大、低层方差小"的梯度
- **Laplace 熵估算差异在千分之一量级**：truck 均匀 1.2833 vs 层级+空间 1.2836，完全可忽略
- **方案B 块内一致性反而更差**：所有通道的 CV(std) 改善均为负数（最差 -129%）
- **原因**：RAHT 的 Haar 变换已经很好地"均质化"了 AC 系数，不同层级的分布趋于一致
- **结论：按八叉树层级重新分块对压缩效率没有帮助，不值得实施**

## 与 FCGS 对比的潜在提升方向分析

### 当前对比状态
- 360: MesonGS+ 赢（低码率明显赢）
- tt: 基本持平，中低码率微输 0.01~0.05 dB
- db: MesonGS+ 全面小赢

### 可行改进方向（按优先级）
1. **ILP 目标函数加 channel importance weight** — 当前所有通道同等 L2 权重，但 opacity/rotation 对渲染重要性更高
2. **percentile 量化（去 outlier）** — 当前 min/max 量化受极端值影响大
3. **Laplace → Gaussian 或自动模型选择** — constriction 支持多种分布，per-block 选最优
4. **Codebook per-SH-dim bit allocation** — 低阶 SH 更重要，应该分配更多 bits
5. **Scaling 做 RAHT 变换** — 当前 scaling 不做 RAHT，可能在 log-space 下可以做
6. **优化 kept point 选择策略** — 改为按 VQ 近似误差最大选择，而非训练时的重要性

## 改进方向消融实验（improvement_ablation）

### 代码修改
- `config.py`: 新增 `channel_importance_weight`, `percentile_quant`, `auto_entropy_model` 三个布尔参数（默认False）
- `model.py __init__()`: 读取并存储三个参数 (~line 441)
- `model.py`: **3处 `search_qbits()` 调用** + **2处 `laplace_encode_blocks()` 调用** 都需要传递新参数
  - `_encode_with_params()` ~line 2112
  - `_get_objective_for_num_keep()` ~line 2230  
  - `encode()` ~line 2763 (实际Budget-First路径!)
- `qbit_search_tool.py search_qbits()`: 
  - channel_importance_weight: 权重 opacity(2.0), quat(1.5×4), features_dc(1.0×3), scaling(0.8×3)
